# OPA Android Demo (前端骨架参考)

这是一个专门为《工科创 II》图像组合与视觉应用项目提供的前端参考骨架。

**⚠️ 重要声明**：
本 Demo **仅仅是一个例子**，主要是为了向大家展示移动端应用的基本交互形态（如图片导入、拖拽交互、以及前景背景合成的可视化界面）。**你不必强行基于此代码进行开发！** 欢迎你们根据团队的技术栈，从零手写自己的 Web 端（Vue/React）、跨平台端（Flutter/React Native）或原生客户端。

---

## 🛠️ 骨架说明 (残缺版)

如果你选择参考本代码或在此基础上进行二次开发，请注意：**本工程内部原有的核心 AI 智能算法（如智能抠图、自动排版推荐、模型评分）已经被故意剥离。**

在 `app/src/main/java/per/jau/opademo/MainActivity.java` 中，请全局搜索 `TODO: [工科创II 学生任务]`，你会看到几处被截断的逻辑。目前点击“自动摆放”只会返回随机分数，你需要发挥主观能动性，将这部分逻辑接管过来。

你有以下**两条主流技术路线**来补全这些功能：

### 路线 A：前后端分离（网络通信）
将 Android 作为纯粹的展示端，把重度的 AI 运算放在你自己的电脑服务端。
* 将合成的图片转为 `Base64` 编码。
* 使用 `HttpURLConnection` 或 `OkHttp` 等网络库，向你编写的 Python 后端（如 FastAPI / Flask）发起 POST 请求。
* 解析服务端传回的打分和推荐坐标。

### 路线 B：端侧本地推理（进阶项）
如果你想挑战“本地推理 (5分)”甚至“MindSpore (5分)”等进阶项，你需要让 AI 跑在手机芯片上。
* 将你微调或导出的模型转换成移动端适配的格式（如 `.onnx`、`.tflite` 或 MindSpore Lite 格式）。
* 在 Android 工程中引入对应的推理运行库（如 `onnxruntime-android`）。
* 在 Java/Kotlin 中将图片转化为 Tensor，喂给本地模型，并直接在手机端获取评分和抠图掩码。

## 📦 编译与运行 (Android Studio)

如果你想把玩一下这个骨架：
1. 下载并安装最新版的 [Android Studio](https://developer.android.com/studio)。
2. 选择 `File -> Open`，定位到本文件夹并打开。
3. 待 Gradle 自动同步完成。
4. 连接你的 Android 手机（需开启开发者模式和 USB 调试），或创建一个虚拟模拟器。
5. 点击顶部的 **Run (绿色三角形)** 按钮，即可看到界面。

**💡 提示**：不要害怕原生开发！无论你遇到网络权限报错、UI 线程阻塞，还是不清楚如何解析图片，都可以把你遇到的情况丢给 AI 代码助手，它们能帮你极快地跨越这道工程屏障。
