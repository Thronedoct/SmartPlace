# SmartPlace API Contract

This document is the stable phase-0 contract between Android and the cloud inference service.

## Base URL

Development:

```text
http://127.0.0.1:8000
```

Android emulator:

```text
http://10.0.2.2:8000
```

Physical Android phone:

```text
http://<computer-lan-ip>:8000
```

## Health Check

```http
GET /api/health
```

Response:

```json
{
  "status": "ok",
  "service": "smartplace-mock",
  "model_version": "mock-v0"
}
```

## Recommend Object Placement

```http
POST /api/place/recommend
Content-Type: multipart/form-data
```

Form fields:

| Field | Type | Required | Description |
|---|---|---:|---|
| `background` | file | yes | Background image selected by the user. |
| `foreground` | file | yes | Foreground object image, preferably PNG with alpha. |
| `mask` | file | no | Foreground mask. If absent, the server will later derive it from alpha or segmentation. |
| `candidate_count` | integer | no | Number of candidates to return. Default: 3. |
| `foreground_scale` | float | no | Approximate foreground width ratio in the background. Default: 0.25. |
| `mode` | string | no | `auto` or `manual`. Default: `auto`. |

Phase-0 response:

```json
{
  "request_id": "mock-20260529-000001",
  "model_version": "mock-v0",
  "coord_type": "normalized_xywh",
  "runtime_ms": 42,
  "image_width": 1,
  "image_height": 1,
  "best_index": 0,
  "candidates": [
    {
      "rank": 1,
      "x": 0.38,
      "y": 0.58,
      "w": 0.22,
      "h": 0.22,
      "score": 0.86,
      "tier": "recommended",
      "label": "цОишНР",
      "reason": "Object is inside a stable support region.",
      "preview_url": null,
      "heatmap_url": null
    }
  ]
}
```

Coordinate rule:

- `coord_type` is `normalized_xywh`.
- `x` and `y` are the top-left corner of the foreground box.
- `w` and `h` are foreground box width and height.
- All values are normalized to the displayed background image, from `0.0` to `1.0`.

Tier mapping:

| Score range | Tier | Chinese label |
|---|---|---|
| `score >= 0.75` | `recommended` | цОишНР |
| `0.45 <= score < 0.75` | `acceptable` | хПпцОехПЧ |
| `score < 0.45` | `rejected` | ф╕НцОишНР |

## Real Model Upgrade Plan

The phase-0 mock response must remain compatible with the later real model response. Later additions may fill:

- `preview_url`: composited candidate image generated by the cloud service.
- `heatmap_url`: Grad-CAM, occlusion, or attention explanation image.
- `model_version`: for example `opa-rgb-v1` or `opa-rgb-mask-v1`.
- `runtime_ms`: measured cloud inference time.
