# 开发协作流程

## 分支约定

- `main`：稳定交付分支。
- `dev`：日常集成分支。
- `feature/model-*`：模型代码、模型改造和实验。
- `feature/backend-*`：云端服务、接口和部署。
- `feature/web-*`：Web 工作台页面、交互和可视化功能。
- `feature/docs-*`：报告、PPT、截图和文档。

## 同步规则

每次小组同步需要回答四个问题：

1. 上次同步后改了什么？
2. 现在有什么内容可以让其他成员运行或 review？
3. 当前卡点是什么？
4. 接口约定是否需要变化？

如果接口字段发生变化，必须先更新 `docs/API.md`，再改 Web 前端或后端代码。

## 合并规则

合并到 `dev` 前：

- 代码能构建，或脚本能在本地启动。
- 行为变化已经写入对应文档。
- 至少一名其他成员看过受影响的接口或使用方式。

合并到 `main` 前：

- Web 工作台能完成主流程。
- 后端日志能显示请求和响应。
- `README.md` 中的运行步骤与当前代码一致。

## 报告分工

- 成员 A 负责模型相关章节。
- 成员 B 负责系统架构、接口、部署和模型调用证据。
- 成员 C 负责 Web 交互、截图、演示视频和最终材料整合。

每个成员先写自己负责部分的初稿。成员 C 可以统一排版，但内容准确性仍由对应负责人保证。
