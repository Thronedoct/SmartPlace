# 高分路线

本文根据 `课程项目.pdf` 重新整理 SmartPlace 的交付路线。结论是：不做 Android 端，采用 Web 应用 + FastAPI 后端 + OPA/libcom 模型适配，优先保证稳定、真实、可解释。

当前短期执行顺序见 `docs/NEXT_ROUTE.md`。

## 课程要求摘要

课程项目的核心要求是做出一个可交互视觉应用，并基于参考模型或参考代码完成必要模型改动。方向 A 的目标是智能物体放置与合成图质量评价，重点是候选位置评分、排序和推荐。

课程允许应用形态为网页应用、手机 App 原型或电脑端应用软件。`OPAAndroidDemoSimp/` 是教学残缺版参考骨架，完全允许不使用。SmartPlace 选择 Web 应用作为主线交付形态。

## 高分目标

SmartPlace 的高分组合：

```text
Web 物体放置工作台
+ FastAPI 本地/局域网推理服务
+ OPA/libcom baseline 真实评分
+ RGB vs RGB+mask 本体类模型改动
+ Top 3 候选生成、评分和排序
+ 0-1 分数与三档标签输出
+ OPA 小子集轻量 fine-tune 或模型改造对比
+ Grad-CAM 或遮挡实验解释
+ 18 组测试案例与成功/失败分析
+ 现场新图片推理日志和权重加载证据
```

不做或不作为主线：

```text
Android App
从零训练大模型
TopNet 训练
端侧手机推理
只展示离线图片而不真实调用模型
```

## 进阶项取舍

优先争取这些进阶分：

| 进阶项 | 最高分 | SmartPlace 方案 | 优先级 |
|---|---:|---|---|
| 模型改造 | 10 | 在基础改动之外，完成候选排序、分数校准、轻量 fine-tune 或 FOPA 候选对比 | P0 |
| 本地推理 | 5 | 模型在本地电脑或局域网 FastAPI 服务中真实推理，不只展示离线结果 | P0 |
| 模型解释 | 10 | 为成功和失败案例生成 Grad-CAM、显著性图或遮挡实验图 | P1 |
| 复杂交互 | 10 | Web 支持候选切换、参数调整、失败提示、结果导出或手动位置评分 | P1 |
| 多模型串联 | 10 | 时间允许时加入自动 mask/抠图、FOPA 热力图或图像协调工具 | P2 |

暂不追求：

- 手机 App 5 分：不做 Android。
- MindSpore/CANN 5+5 分：除非后续有稳定环境和足够时间，否则不作为主线。

## 端到端主链路

1. 用户在 Web 工作台上传背景图和前景图。
2. FastAPI 接收图片，生成候选位置和尺度。
3. 后端合成每个候选图，并生成或读取前景 mask。
4. OPA/libcom baseline 或改造模型输出每个候选的合理性分数。
5. 后端排序并返回 Top 3、三档标签、模型版本、耗时、候选预览图和可选解释图。
6. Web 展示推荐框、分数、候选详情和运行证据字段。
7. 报告记录成功案例、失败案例、模型改动、运行日志和 AI 辅助说明。

## 最小可行验收

第十五周课堂汇报前至少完成：

- Web 可以上传一组全新背景/前景图。
- 后端终端能打印请求、候选数量、模型版本和推理耗时。
- 模型不是 mock，能展示权重加载位置和张量推理代码段。
- 前端展示 Top 3 候选框、分数、三档标签和 `model_version`。
- 至少 6 组真实案例，包含效果好和效果差的例子。
- PPT 中能说明参考代码输入输出、模型改动文件、改动原因和限制。
